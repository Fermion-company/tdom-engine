# 開発ロードマップ — 第一目標「canonicalが体感から消えたリアルタイムプレビュー」

**確定日: 2026-07-07（ユーザー決定）**

目標は「canonical なしのリアルタイムプレビュー」では**ない**。canonical
（本物の LuaLaTeX ラン）は正しさの唯一の根拠として恒久的に残す。目標は、
その存在が**ユーザーの体感から消える**こと:

> キーを打つ → 全ページが即座に更新（ms）→ 見ているページに本物の確定が
> 来る（〜0.1 秒）→ 確定の波が下流へ広がる → バッジが「exact」に戻る。
> opaque は実文書で発火せず、「コンパイルを待つ」という体験自体が存在しない。

測定可能な達成条件:

- **A. 権威の追いつき**: ポーズ後、閲覧中ページの exact 確定 ≤ 0.3 秒
  （70p 和文ストレス文書）。全文converge時間は編集点以降のページ数に比例。
- **B. 打鍵応答**: 打鍵→ページ更新 p95 ≤ 100ms（巨大段落を含む）。
- **C. opaque 発火率**: 実文書コーパスで ≈ 0。
- **D. 正しさ**: 「増分＝ゼロから」の等式がコーパス＋編集ファザーで恒常成立。

## Phase 0 — 監査装置（他フェーズの前提。権威層を触る前に比較器を持つ）

1. **コーパス差分農場**: 実文書 N 本で「収束後表示 = 素の lualatex」を
   ピクセル比較（pdftoppm）。副産物: JS ページビルダーの改ページ一致率、
   opaque 発火率、fidelity ティア分布の各メトリクス。
2. **編集ファザー**: ランダム編集列 → 毎回「収束後 = その時点ソースの
   コールドコンパイル」を機械検証（tests/hot-path.test.js の等式テストの
   一般化）。増分バグは編集順依存で出るため、これが唯一の網。
3. **レイテンシベンチ**: 打鍵レイテンシ percentile を文書別に記録する
   スクリプト（scratch の probe-typing.mjs を tools/ へ昇格）。

完了条件: `npm run farm` / `npm run fuzz` が回り、メトリクスが数値で出る。

## Phase 1 — incremental canonical（権威の 0.1 秒化。本丸）

本物の output routine を生かした第二の常駐ラン（shipping chain）を、
`\shipout` ごと（ページ境界）に fork で凍結する。編集 → 編集ページ直前の
checkpoint から再開 → 以降のページだけ re-ship。再開後も「同じランの続き」
なので出力は再実装ゼロの本物。

1. **1a. daemon**: 実ページビルダー稼働モード＋ shipout 境界での fork
   checkpoint（dormant chain とは別プロセスツリー）。
2. **1b. orchestrator**: shipping chain のライフサイクル（編集→ページ k で
   切断→再開→ページ着地イベント）。既存 CanonicalRenderer は「コールド
   最終監査」として残す（従量制のまま、頻度を大きく下げる）。
3. **1c. 後方波及の検出**: aux/label 差分で「編集が前のページに影響」を
   検知したらコールド再コンパイルへフォールバック（率を農場で計測）。
4. **1d. クライアント**: ページ単位の canonical 着地イベント（既存の
   canonical SSE を部分範囲対応に拡張）。閲覧中ページ優先の SVG 変換は
   既存の viewport-aware 機構を流用。
5. **1e. 資源方針**: ページ checkpoint の間引き（K ページごと）、RSS 実測。

前提計測: 編集局所性（後方波及率）・ページ checkpoint あたり RSS。
完了条件: 達成条件 A。

## Phase 2 — 「見える依存」の残りを消す

- **2a. 巨大段落の打鍵コスト**（現状: 10 行超 CJK 段落で 1 打鍵 200ms 台）。
  候補: (i) 編集ブロックも stale-first にして応答を非同期化、(ii) 段落
  ジョブのプロファイル（luahbtex シェーピングの内訳）、(iii) 表示専用の
  段落分割。※段落は Knuth-Plass の原子なのでソース分割は意味論が変わる
  ——表示層での対処が原則。
- **2b. opaque リスト各個撃破**（易→難）: `\enlargethispage`（ページ goal
  調整1行）→ `\marginpar/\marginnote`（canonical-only ブロック tier か
  余白込みチャンク）→ eso-pic/透かし（ページ装飾チャンク層: ページ
  スタイルごとに1回描いて下敷き合成）→ `\newgeometry` → twocolumn
  （ltoutput 2 段組の転写。最大）。生の `\output` 書き換えだけは opaque の
  まま残す（quirks mode としての存在意義）。
- **2c. Stage B 伝播**（docs/10）: counter 差分の選択的伝播で settle パスを
  O(影響ブロック) に。
- **2d. toc 経路依存の解消**（増分=ゼロから の残り 1/285: `\tableofcontents`
  ブロック）。

完了条件: 達成条件 B・C。

## Phase 3 — 仕上げ

- **ストリーミング boot**: open で最初のページ群を数秒で表示し、残りは
  背景で継続（現状 70p 文書で約 2 分の空白）。
- 長文書のメモリチューニング（checkpoint 密度の適応化）。
- 監査農場の CI 常設化。

## 恒久ドクトリン（全フェーズで不変）

1. canonical は最終権威。消すのではなく**見えなくする**。
2. fidelity gate の既定は常に下へ（速くて間違う表示は唯一の死因）。
3. ソースが唯一の真実（表示からの逆変換は作らない）。
4. エンジン（LuaTeX バイナリ）はフォークしない。触るなら小さなフックのみ。
5. foreground は有界（docs/10 §I1）。アイドル時は計算ゼロ（§I3）。
6. 権威層に触る変更は、比較器（Phase 0）が先に存在すること。
